# 数据库表结构

本文档给技术和 AI 阅读，方便代码审查和 Git diff。业务核对字段时优先查看 `docs/数据库表结构.xlsx`。

## 维护规则

- SQL 初始化源头：`backend/db/init/001_schema.sql`
- 已有数据库升级：`backend/db/migrations/*.sql`
- 技术/AI 文档：`docs/数据库表结构.md`
- 业务核对表：`docs/数据库表结构.xlsx`

新增或修改表结构时，以上四处必须同步。不能只改 SQL，也不能只改 Excel。

## 设计约定

- 数据库：PostgreSQL
- 表名前缀：`pms_`
- 主键：默认使用 `id BIGSERIAL PRIMARY KEY`
- 删除标记：业务主表默认使用 `is_deleted SMALLINT`，`0` 正常，`1` 删除
- 时间字段：统一使用 `TIMESTAMPTZ`，默认 `NOW()`
- 接口返回：后端字段保持数据库命名，前端 API 层转换为页面 record

## pms_bug BUG 表

| 字段 | 类型 | 约束/默认值 | 说明 |
|---|---|---|---|
| `id` | `BIGSERIAL` | 主键 | BUG 编号 |
| `source_type` | `SMALLINT` | 必填；1 项目，2 需求 | 关联类型 |
| `project_id` | `BIGINT` | 可空，引用 `pms_project.id` | 关联项目 |
| `requirement_id` | `BIGINT` | 可空，引用 `pms_requirement.id` | 关联需求 |
| `title` | `VARCHAR(200)` | 必填，正常数据中唯一 | Bug 标题 |
| `description` | `TEXT` | 可空 | Bug 描述（富文本） |
| `bug_type_id` | `BIGINT` | 必填，引用 `pms_archive.id` | Bug 类型，档案类型为“Bug类型” |
| `severity` | `SMALLINT` | 必填 | 严重程度：1 低，2 中，3 高，4 致命 |
| `status` | `SMALLINT` | 默认 `0` | 0 新建，1 已修复，2 已关闭，3 被激活 |
| `assignee_id` | `BIGINT` | 必填，引用 `pms_user.id` | 指派给 |

| `closed_date` | `DATE` | 可空 | 关闭时间 |
| `activation_reason` | `TEXT` | 被激活时必填，最多 100 字 | 最近一次激活原因；后续修复或关闭时继续保留，再次激活时更新 |
| `creator_id` | `BIGINT` | 引用 `pms_user.id` | 创建人 |
| `updater_id` | `BIGINT` | 引用 `pms_user.id` | 更新人 |
| `is_deleted` | `SMALLINT` | 默认 `0` | 删除标记 |
| `created_at` | `TIMESTAMPTZ` | 默认 `NOW()` | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | 默认 `NOW()` | 更新时间 |

关联项目和关联需求必须按关联类型二选一；“已修复”“已关闭”和“被激活”状态可保留解决方案与修复时间，被激活后再次修复时使用新值覆盖；只有“已关闭”状态保留关闭时间；“被激活”状态必须填写激活原因，后续修复或关闭后继续保留最近一次激活原因。索引：`uk_bug_title_active`、`idx_bug_project`、`idx_bug_requirement`、`idx_bug_assignee_status`、`idx_bug_type_status`、`idx_bug_severity_status`、`idx_bug_created_at`。
