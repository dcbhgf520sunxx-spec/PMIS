# 任务详情返回语义与子任务完成时间设计

## 目标

1. 主任务详情的子任务表格在“预计完成时间”后展示“实际完成时间”。
2. 从主任务详情进入子任务详情时，返回按钮显示“返回”，并返回主任务详情。
3. 从任务列表进入任务详情时，返回按钮继续显示“返回列表”。

## 方案

- 复用 `TaskRecord.actualEndTime`，不新增接口、字段或数据库迁移。
- `usePageReturnNavigation` 已提供校验后的 `returnTarget`。任务详情根据该目标是否为同模块任务详情地址，决定传给 `TemplateDetailPage.backText` 的文案。
- 只有返回目标匹配 `/tasks/:id` 时使用“返回”；返回目标为 `/tasks` 或带列表查询参数的 `/tasks?...` 时使用默认“返回列表”。
- 返回动作仍统一调用 `returnToSource`，不新增硬编码跳转。

## 页面变化

- 子任务表格列顺序：任务名称、负责人、任务类型、优先级、状态、预计完成时间、实际完成时间、操作。
- 实际完成时间为空时显示 `-`。
- 子任务详情从主任务详情进入时，右上角按钮显示“返回”。

## 验证

- 自动测试覆盖两种返回来源的文案判断。
- 自动测试覆盖子任务表格“实际完成时间”列及顺序。
- 运行 `node scripts/verify-change.mjs`。
- 浏览器分别验证主任务详情子任务表格、主任务详情进入子任务详情、任务列表进入任务详情。

## 边界

- 不修改其他模块详情页的默认“返回列表”文案。
- 不修改任务接口、数据库结构、状态流转和返回目标解析规则。
