# 任务管理迁移设计

## 范围

将 `project-manage-system` 的任务管理完整迁入 PMIS，只迁移功能，不迁移任何历史数据，不改首页统计，不接入源项目 MCP。业务字段、状态和操作规则沿用源系统；页面、接口、权限、日志和数据库实现遵守 PMIS 当前 React、Express、PostgreSQL、页面样板和公共组件规范。

## 数据模型

新增 `pms_task` 表，包含任务名称、描述、来源类型、关联项目、关联需求、负责人、任务类型、优先级、状态、逾期标记、启动时间、预计完成时间、实际完成时间、暂停时间、创建更新人、软删除和时间戳。任务来源只能是项目或需求之一；未删除任务名称唯一；外键关联项目、需求、用户和任务类型档案；常用筛选、排序及关联字段建立索引。历史记录复用 `pms_op_log.operation_id` 聚合，不新增历史表。

数据库物理字段继续沿用 `start_date`、`expected_end_date`、`actual_end_date`、`suspend_date`，避免无业务收益的结构改名；页面标签、提示、历史字段、测试名称和交付材料统一使用“时间”，不出现“日期”。

初始化数据只增加“任务类型”档案类型及源系统对应的任务类型选项，不插入任何任务记录。增量迁移执行前需再次确认本设计已经获批。

## 业务规则

- 状态保持为未启动、进行中、已完成、暂停。
- 未启动可进入进行中或暂停；进行中可进入已完成或暂停；已完成可进入暂停；暂停仅恢复到暂停前状态。
- 进入已完成时填写实际完成时间，进入暂停时填写暂停时间；离开对应状态时按源系统规则清理或保留关联字段。
- 预计完成时间早于当天且任务未完成、未暂停时标记逾期。
- 任务必须关联一个有效项目或一个有效需求，二者不能同时选择。
- 负责人必须是有效用户；任务类型必须来自有效的“任务类型”基础档案。
- 未删除任务名称不可重复。
- 项目、需求或任务类型档案仍被未删除任务引用时禁止删除，并返回明确的引用数量。
- 创建人、更新人和日志操作人只从当前登录用户取得，前端不能提交或覆盖操作者身份。

## 页面设计

新增一级菜单“任务管理”，地址 `/tasks`，菜单和接口独立授权，不调整首页及其他菜单业务。

列表页使用 `TemplateListPage` 和 `useTemplateListPageData` 的服务端模式，提供“全部”和“我的”视图及真实数量统计。筛选覆盖任务名称、关联来源、项目或需求、任务类型、优先级、状态、逾期、负责人和预计完成时间；表格支持统一排序、分页、固定列和列宽设置。第一列进入详情，操作列使用 `OperationColumnActions`，包含编辑、复制、指派、状态变更和删除，并按公共组件规则收纳到“更多”。

新增、编辑和复制页使用 `TemplateFormPage`、`TemplateFormSection` 及现有 `AdminProForm*` 控件。复制带入可复用的业务字段，但不带任务编号、状态、实际完成时间、暂停时间和历史记录，保存后生成新任务。

详情页使用 `TemplateDetailPage`。状态、优先级和逾期标签通过 `statusSection.items` 同步展示在标题和右侧；状态操作通过任务业务封装直接渲染公共 `StatusChangeAction`。基础信息展示关联来源、负责人、任务类型和各时间字段；变更历史使用 `HistoryTimelineSection`；上一条、下一条复用详情模板现有邻接导航能力，并保持列表筛选和排序上下文。

## 接口与数据流

新增 `/api/tasks` 接口组，覆盖列表、详情、新增、编辑、删除、状态变更、名称校验、历史记录和上下条查询。前端 API 层将后端字段转换成页面记录，并为读写返回值声明运行时契约。所有接口使用统一 `{ code, message, data }` 响应；名称重复、来源缺失等可归属字段的错误使用结构化 `fieldErrors` 回填表单。

列表统计、数据列表和上下条查询共享同一套筛选与排序规则。“我的”范围使用当前登录用户，不与负责人筛选参数混用。状态合法性、附加时间字段和逾期计算由后端独立校验，前端仅展示当前允许的目标状态。

## 权限与关联保护

同步前端路由、侧栏菜单、后端路由挂载、初始化菜单及增量迁移。`/tasks` 是任务管理权限键，新增、复制、编辑和详情子路由继承该权限；未授权用户无菜单入口，直接访问页面或接口均被拒绝。

项目、需求和基础档案删除接口增加任务引用检查，只增加与任务迁移直接相关的保护，不改其余业务规则。

## 验证与交付

自动测试覆盖项目/需求二选一、名称唯一、操作者身份、合法与非法状态流转、完成和暂停时间、逾期计算、复制、列表统计、筛选排序分页、上下条、历史聚合、权限拒绝及关联删除保护。时间术语门禁扩展到任务模块。

完成实现后运行 `node scripts/verify-change.mjs` 和数据库迁移检查。用户已确认本表结构后才执行 migration，再验证真实接口读写。浏览器实际检查 `3104` 的列表、查询、重置、排序、分页、空状态、新增、编辑、复制、详情、状态变更、删除、历史、上下条和权限；同时确认 `3103` 健康检查返回 200，前端 `3104` 可访问。

首页统计、MCP、源系统数据导入以及任务业务规则调整均不在本次范围内。
