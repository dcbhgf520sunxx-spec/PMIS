# 需求复制设计

## 目标

在需求列表操作列中增加“复制”，复用现有新增需求表单创建一条新需求。

## 方案

- 操作顺序保持“编辑、状态变更、复制、删除”；4 个动作由 `OperationColumnActions` 自动将第 3 个起收入“更多”。
- 新增 `/requirements/:id/copy` 路由，仍使用 `RequirementFormPage`，通过 `mode="copy"` 区分复制场景。
- 复制页读取源需求，回填标题、描述、需求路径、产品、项目、负责人、优先级、提出信息和计划时间。
- 保存时调用现有 `createRequirement`，由后端生成初始状态、编号、创建人和创建时间；不复制源需求的状态、实际完成时间、暂停时间和变更历史。
- 标题原样回填。若未修改标题，现有后端字段级唯一校验在“需求标题”下提示“需求标题已存在”。
- 保存或取消后，通过现有 `returnTo` 链路返回原列表状态。

## 验收

- 列表操作出现“复制”，点击后进入“复制需求”。
- 表单正确回填源需求可复制字段。
- 未修改重复标题时显示字段错误；修改标题后能创建新需求并返回原列表。
- 无数据库结构、接口、菜单和权限变化。
