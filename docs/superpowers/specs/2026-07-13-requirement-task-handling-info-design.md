# 需求与任务处理信息归类设计

## 目标

统一需求、任务新增编辑页与详情页的字段分组，把“负责人”放到“处理信息”第一项，并让任务的处理类时间字段集中展示。

## 字段归类

- 需求基本信息：需求标题、需求描述、需求路径、所属产品、所属项目、优先级。
- 需求处理信息：负责人、提出人、提出组织、提出时间、启动时间、预计完成时间；详情页继续展示实际完成时间、暂停时间、完成情况。
- 任务基本信息：任务名称、任务描述、关联类型、关联对象、任务类型、优先级。
- 任务处理信息：负责人、启动时间、预计完成时间；详情页继续展示实际完成时间、暂停时间。

## 实现约束

- 继续使用 `TemplateFormSection`、`TemplateDetailSection`、`DetailMetaList` 和已有 `AdminProForm*` 控件。
- 新增与编辑共用现有表单页面，不新增组件，不修改接口、字段和数据库。
- 仅调整字段所在分组与展示顺序，不改变校验、取值、提交和状态流转逻辑。

## 验收

- 需求和任务表单均存在“基本信息”“处理信息”两个分组，负责人是处理信息第一个字段。
- 需求和任务详情页的负责人均从基本信息移入处理信息第一项。
- 任务的启动时间、预计完成时间、实际完成时间、暂停时间均位于处理信息。
- 结构测试、完整门禁、浏览器新增页和详情页检查通过。
